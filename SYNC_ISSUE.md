# 訂單同步問題說明

## 問題
手機點餐後，電腦後台看不到新訂單。

## 原因
localStorage 是基於瀏覽器的本地儲存，不同裝置的瀏覽器各自有獨立的 localStorage：
- 📱 手機瀏覽器 → 獨立的 localStorage
- 🖥️ 電腦瀏覽器 → 獨立的 localStorage
- ❌ 兩者不會自動同步

## 目前的暫時測試方法

### 方法 1：同一瀏覽器測試
1. 在電腦開啟：`http://192.168.18.3:5173/admin/orders`
2. 開新分頁：`http://192.168.18.3:5173/order?table=5`
3. 在新分頁點餐送出
4. 切回訂單頁面，2秒內會自動同步

### 方法 2：手機測試後手動重新整理
1. 手機掃 QR Code 點餐
2. 電腦後台**手動重新整理頁面** (F5)
3. 訂單會出現

## 長期解決方案

需要實作以下其中一種：

### 選項 A：使用後端 API + 資料庫（推薦）
- 建立後端伺服器（Node.js + Express）
- 使用資料庫儲存訂單（SQLite / MongoDB）
- 所有裝置都讀寫同一個資料庫
- 可加入 WebSocket 實現即時同步

### 選項 B：使用雲端服務（快速方案）
- Firebase Realtime Database
- Supabase
- 免費且即時同步

### 選項 C：使用本地檔案共享（僅限同網域）
- 實作簡單的 JSON 檔案 API
- 所有裝置讀寫同一個 JSON 檔案
